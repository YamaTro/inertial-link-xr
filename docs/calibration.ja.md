# 較正と軸の確認

較正は静止時のセンサーバイアスを求め、明示的なスマホ取付方向を確認する作業です。快適な表示を自動決定したり、有効性を証明したりするものではありません。

通信座標は右手系で、+Xが右、+Yが上、-Zが前です。既定の`SCREEN_UP_TOP_FORWARD`は、画面を上、スマホ上端を車両前方へ向け、Android `(x,y,z)`を`(x,z,-y)`へ変換します。他の取付方向では専用の正しい回転変換とテストが必要です。

## 静止較正

1. ゴーグルを誰も装着していない状態で行います。
2. スマホを最終的な向きで、完全に静止した硬い面・停車車両へ固定します。
3. セッションごとの新しい鍵で接続し、「Stationary calibration」を開始します。
4. 完了するまで触れません。移動を検出したら失敗させ、動いた値を採用してはいけません。
5. `CALIBRATED`が有効、`CALIBRATING`が無効になり、`calibrationId`が変化したことを確認します。
6. 静止中の補正済み角速度・線形加速度がゼロ付近で、一定方向へ流れ続けないことを確認します。

固定位置・端末を変えた場合、継続した偏りがある場合は停車中に再較正します。走行中には行いません。

## 軸と符号

数値診断または大きく分かるテスト物体を使い、未装着で行います。右、上、前へ1軸ずつ短く動かし、期待する+X、+Y、-Zだけが主に反応することを確認します。各軸回転、合成`gentle-turn`、誤った鍵、重複・遅延パケット、送信停止も試し、不正データで動きが維持されないことを確認します。

視覚表現のゲインは別の較正です。ゼロから始め、1項目ずつ変更し、上限と即時停止を維持してください。詳しい記録項目と注意は[英語版](calibration.md)を参照してください。
